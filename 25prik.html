25.Приклад програми з коментарями

Снова возвращаемся к примеру из вопроса 2.1:

Построчное описание программы.

TITLE Add and Subtract (AddSub.asm)

Директива TITLE по сути является строкой комментария, в которую можно пoместить любой текст. Обычно после этой директивы помещается название программы.

; This program adds and subtracts 32-bit integers.

Текст, расположенный после символа точки с запятой, является комментарием и поэтому игнорируется компилятором. Обычно в комментарии, расположенном после директивы TITLE, приводится краткое описание программы.

INCLUDE Irvine32.inc

С помощью директивы INCLUDE в программу можно включить информацию (такую как определения глобальных переменных и начальные установки программы), расположенную в указанном файле (в данном случае irvine32. inc). По умолчанию считается, что включаемые файлы расположены в подкаталоге INCLUDE, расположенном в каталоге, в который установлен ассемблер.

.code

Директива .code обозначает начало сегмента кода, в котором размещаются все команды программы, выполняемые процессором.

main PROC

С помощью директивы PROC в программе обозначается начало процедуры. Вместо имени процедуры main могло быть использовано любое другое имя.

mov eax,10000h

Команда MOV загружает в регистр EAX целое число 10000h. Ее первый операнд (EAX) называется получателем, а второй операнд — источником.

add eax,40000h

Команда ADD прибавляет к содержимому регистра EAX число A0000h.

sub eax,20000h

Команда SUB вычитает из регистра EAX число 20000h.

call DumpRegs

Команда CALL вызывает процедуру, которая отображает текущее значение регистров процессора. Она очень полезна при отладке программ и позволяет проверить корректность работы программы.

еxit

Оператор exit неявно вызывает стандартную функцию системы Windows, с помощью которой завершается выполнение программы. Этот оператор определен в файле lrvine32.inc.

main ENDP

Директива ENDP отмечает конец процедуры.

END main

Директива END отмечает последнюю строку программы, которая будет обработана ассемблером. Кроме того, в ней указывается имя точки входа в программу (т.е. адрес, по которому операционная система передаст управление программе при ее запуске).
